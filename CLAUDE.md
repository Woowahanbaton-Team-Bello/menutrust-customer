# CLAUDE.md

이 파일은 Claude Code가 이 저장소에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

**menutrust-customer**는 식당·카페 손님을 위한 알레르기 메뉴 확인 웹 서비스입니다.
손님은 앱 설치 없이 매장 QR을 스캔해 웹 메뉴판에 접속하고, 본인의 알레르기를
선택하면 각 메뉴가 자신의 알레르기 기준으로 어떻게 분류되는지 확인할 수 있습니다.

이 저장소는 **손님(customer)용 프론트엔드**입니다. 매장/메뉴 데이터를 관리하는
관리자 측은 별도이며 이 저장소 범위 밖입니다.

### 사용자 흐름

1. 손님이 매장 QR을 스캔한다.
2. 메뉴판 웹에 접속한다.
3. 손님이 본인의 알레르기를 여러 개 선택한다.
4. 선택한 알레르기를 기준으로 메뉴가 분류된다.
5. 메뉴가 **확정 / 혼입 가능 / 없음** 상태로 표시된다.
6. 손님은 각 메뉴가 왜 그 상태인지 이유를 확인한다.
7. 손님은 이 정보를 참고해 주문 여부를 판단한다.

## 핵심 도메인 개념

### 알레르기 분류 상태 (3단계)

선택한 알레르겐 기준으로 각 메뉴는 다음 셋 중 하나로 표시됩니다.

- **확정**: 해당 알레르겐이 메뉴에 확실히 포함됨 (원재료로 사용).
- **혼입 가능**: 원재료는 아니지만 조리 과정 등에서 섞여 들어갈 수 있음.
- **없음**: 해당 알레르겐이 포함되지 않음.

각 상태에는 **이유(reason)**가 함께 표시되어야 합니다. 손님이 상태만 보는 게
아니라 "왜 이 상태인지"를 확인하고 스스로 주문 여부를 판단하는 것이 이 서비스의
핵심 가치입니다.

### 알레르겐 목록 (22종)

손님이 선택할 수 있는 알레르겐은 다음 22가지입니다 (한국 식약처 표시 대상 기준):

알류(가금류), 우유, 메밀, 땅콩, 대두, 밀, 고등어, 게, 새우, 돼지고기, 복숭아, 토마토,
아황산류, 호두, 닭고기, 쇠고기, 오징어, 조개류, 굴, 전복, 홍합, 잣

> 이 목록은 도메인 상수입니다. 코드에서 하드코딩을 흩뿌리지 말고 한 곳에서
> 정의해 재사용하세요.

### 데모 매장

현재 데모 매장은 **타이(태국) 음식점**입니다. 샘플 메뉴 데이터는 타이 음식점을
기준으로 구성합니다.

## UI / UX 원칙

- **모바일 우선**: 웹이지만 손님은 QR로 접속하므로 사실상 100% 모바일에서
  봅니다. 모든 화면은 모바일 세로 화면에서 보기 좋게 설계합니다.
- **터치 친화적**: 알레르기 선택 등은 손가락으로 누르기 쉬운 크기와 간격을
  유지합니다.
- **상태 가독성**: 확정 / 혼입 가능 / 없음 3단계는 색·아이콘 등으로 즉시
  구분되도록 하되, 색에만 의존하지 않고 텍스트/아이콘도 함께 제공합니다
  (접근성).
- **빠른 판단**: 손님이 주문 여부를 빠르게 판단할 수 있도록 정보를 간결하게
  보여줍니다.

## 기술 스택

- **React 19** + **Vite** (`@vitejs/plugin-react`)
- 빌드 도구: Vite
- 린터: **oxlint** (`.oxlintrc.json`)
- 언어: JavaScript (JSX), ESM (`"type": "module"`)

## 명령어

```bash
npm run dev      # 개발 서버 (Vite, HMR)
npm run build    # 프로덕션 빌드
npm run preview  # 빌드 결과 미리보기
npm run lint     # oxlint 실행
```

## 프로젝트 구조

```
index.html          # 진입 HTML
src/
  main.jsx          # React 앱 부트스트랩 (StrictMode)
  App.jsx           # 루트 컴포넌트
  App.css
  index.css
  assets/           # 이미지 등 정적 자원
public/             # 그대로 서빙되는 정적 파일 (icons.svg 등)
```

> 현재 `src/App.jsx`는 Vite 기본 스캐폴드 상태입니다. 실제 서비스 화면을
> 구현하면서 교체해 나갑니다.

## 작업 시 주의사항

- 알레르겐 22종 목록과 3단계 상태값은 도메인 상수이므로 단일 소스에서 관리합니다.
- 새 화면·컴포넌트는 모바일 세로 레이아웃을 먼저 고려해 작성합니다.
- 상태(확정/혼입 가능/없음)를 표시할 때는 항상 그 **이유**를 함께 보여줄 수 있는
  구조로 만듭니다.
